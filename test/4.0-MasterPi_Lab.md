# Lab 4.0 – MasterPi AprilTags Navigator

## Overview
Use computer vision to detect AprilTags and steer the MasterPi robot through a course of waypoints.

## Learning objectives
- Capture frames from the Pi camera with OpenCV
- Detect AprilTags with `pupil‑apriltags`
- Convert tag pose to heading & distance
- Command mecanum wheels to strafe toward target
- Chain multiple tags for a full route

## Prerequisites
- Lab 3 completed
- MasterPi hardware assembled
- `opencv‑python` and `pupil‑apriltags` installed (see **Guides/MasterPi_AprilTags_Guide.md**)

## Starter code (Python 3)

```python
import cv2
from pupil_apriltags import Detector
from masterpi import chassis              # provided API

AT = Detector(families="tag36h11")
TARGET_ID = 0
LIN_SPEED = 0.2     # m/s
ANG_SPEED = 0.4     # rad/s
DIST_KP   = 0.5     # PID gains, use simple P for now
ANGLE_KP  = 1.2

def center_tag(frame, tag):
    # Calculate pixel‑space offset
    h, w = frame.shape[:2]
    cx, cy = tag.center
    dx = (cx - w/2) / (w/2)   # ‑1..1
    return dx

def main():
    cam = cv2.VideoCapture(0)
    while True:
        ret, frame = cam.read()
        if not ret:
            break

        tags = AT.detect(frame, estimate_tag_pose=False)

        if tags:
            tag = next((t for t in tags if t.tag_id == TARGET_ID), None)
            if tag:
                offset = center_tag(frame, tag)
                chassis.strafe(-offset * ANG_SPEED)     # side‑to‑side
                # TODO: estimate distance from tag.corners and drive forward

        cv2.imshow("view", frame)
        if cv2.waitKey(1) & 0xFF == ord('q'):
            break

    cam.release()
    cv2.destroyAllWindows()

if __name__ == "__main__":
    main()
```
